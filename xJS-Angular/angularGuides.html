1. Install TypeScrypt and Angular CLI

  1. For using Angular it is a good practice to combine it with using TypeScript. To install TypeScript open command prompt and write the following npm install -g typescript and press Enter.

    Note: This step has to be done only once and it will works for all projects because we are installing it with global –g modifier which will install in the folder with all modules and not in the current project folder

  2. The next step is to install Angular CLI which will allow us to use commands for working with Angular. To install Angular CLI in the command prompt write the following npm install -g @angular/cli and press Enter

    Note: This step has to be done only once and it will works for all projects because we are installing it with global –g modifier which will install in the folder with all modules and not in the current project folder

2. Creating new Angular project

  1. In the command prompt open the folder where you want to start the new project. And then type ng new example-project and press Enter

    Note: example-project is an example folder name where our project is created, you can specify another one if you want. Only ng new has to be the same

  2. Open the new created project with Visual Studio Code and you are ready to start working on the project.

3. Creating Component

We can create our own new component in 2 ways: manually or using the angular cli. First we will see how to create our own new component manually.

3.1 Creating Component manually

  1. The app folder in src folder is where our components are stored

    We can see our main components holding the start point of project which is app.component.xxx

        src
        app 
            app.component.css
            app.component.html
            app.component.specs.ts
            app.component.ts
            app.module.ts

  2. We will create another folder named games in the app folder where we will store our new component. And in the new folder we will create our first needed file games.component.ts which is the typescrypt component

    Note: Folder name “games” is example name, of course we can name our folder the way we want to easier recognize it. Same applies for the file games.component.ts where only the games part is optional and then we must write .component.ts

    If we have done everything correctly our directory should look like: 

        src
        app 
            games
                games.component.ts 
            app.component.css
            app.component.html
            app.component.specs.ts
            app.component.ts
            app.module.ts

  3. We will also create the html and css component files in the folder games which is where our new component is placed. And after we are done our directory should look like this:

        src
        app 
            games
                games.component.css
                games.component.html
                games.component.ts
            app.component.css
            app.component.html
            app.component.specs.ts
            app.component.ts
            app.module.ts

  4. Now we will add content to the file games.component.ts First we will import Component module to be able to use it. Every class component must has @Component decorator function in which we specify 3 items: The first one is selector which specify in which html element component will be rendered. Template pointing which html file will be our template. And the last property styleUrls is an array with all stylesheets we are going to use.

    File: games.component.ts

import { Component } from '@angular/core';

@Component(
    {
        selector: 'game',
        templateUrl: './game.component.html',
        styleUrls: [ './game.component.css' ]
    }
)
export class GameComponent {
    title: string = "Assassin's Creed"
}

  5. We have created our component but the starting point of the project doesn’t know yet so we have to specify it in the app.module.ts file. Before we write anything in it by default it looks like this:

    File: app.module.ts
                    
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';

import { AppComponent } from './app.component';

@NgModule({
    declarations: [
        AppComponent
    ],
    imports [
        BrowserModule
    ],
    providers: [],
    bootstrap: [AppComponent]
})
export class AppModule { }

    Now lets tell angular about our new component by importing it and also specify it in the declarations property

    File: app.module.ts
                                        
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
                    
import { AppComponent } from './app.component';
import { GameComponent } from './games/games.component';
                    
@NgModule({
    declarations: [
        AppComponent,
        GameComponent
    ],
    imports [
        BrowserModule
    ],
    providers: [],
    bootstrap: [AppComponent]
})
export class AppModule { }

    We have our component ready but we still need to add content to the html file.

  6. In our ts component file we have specified property title so lets use it in the html. We have only one h1 tag holding the title property from ts file. Using double curly brackets like this is called data binding, meaning that we get data from the ts file and use it in our html template

    File: games.component.html
            
    <h1>{{title}}</h1>

  7. The next step is to use the selector from our ts component file in the parent html template where we want our component to be rendered. In our ts file we named our selector 'game' so to render it we need to name the html element 'game'. We will render it in the parent html file which is app.component.html We will remove some of the html elements from it to make it more simple looking and make it look like this:

    File: app.component.html
                                
    <div style="text-align:center">
        <h1>
            Welcome to {{title}}!
        </h1>
        <img width="300" alt= .....>
    </div>
    <game></game>

  8. And now to display everything we have created so far let's run the project. We do that by typing ng serve --open

    If we have done everything correctly so far in the broswer we should see the following screen

    Note: --open part of the command is to also open the project in the browser, without it we have to open it manually usually on localhost:4200

  1. To create component with the CLI go to the Visual studio code terminal and type ng generate component about and press Enter. Keep in mind only ng generate component has to be the same and "about" is random name of how we want to name our new component

    And that's it, the angular CLI created for us html, css and ts component files plus specs file used for unit testing. And also imported our new module in the app.module.ts file
